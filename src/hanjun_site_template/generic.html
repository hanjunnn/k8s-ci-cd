What's Kubernetes?

쿠버네티스(Kubernetes)는 컨테이너화된 애플리케이션의 배포, 확장, 관리 자동화를 돕는 오픈소스 시스템입니다.
클러스터 환경에서 애플리케이션을 안정적으로 운영할 수 있도록 지원하며, 고가용성 및 자동화된 배포 기능을 제공합니다.


이 사이트는 쿠버네티스를 통해 배포되었으며, 로컬PC에 VM 환경을 구축하여 쿠버네티스를 배포하고 프로젝트를 구현했습니다.
실습 과정에서는 Nginx를 웹서버로 사용하고, Istio Ingress Gateway를 활용해 외부 트래픽을 관리했으며,
Kiali를 통해 네트워크 상태를 모니터링했습니다. 또한, Rancher를 사용해 쿠버네티스 클러스터를 관리하고,
GitHub, Docker Hub, Jenkins, ArgoCD와 같은 다양한 도구들을 통합하여
자동화된 CI/CD 파이프라인을 구축하고, 애플리케이션의 지속적인 통합 및 배포를 구현했습니다.

(CI/CD Flow)

Nginx

Nginx는 고성능 웹 서버이자 리버스 프록시 서버로, 효율적인 HTTP 요청 처리와 로드 밸런싱, 캐싱 기능을 통해 시스템 성능을 향상시킵니다. 높은 트래픽 환경에서도 안정적인 서비스를 제공하며, 대규모 애플리케이션의 요구 사항을 충족시킵니다.

Rancher

Rancher는 Kubernetes 클러스터를 관리하고 오케스트레이션하는 오픈소스 플랫폼입니다. 이 플랫폼은 여러 클러스터를 중앙에서 관리하고, 보안, 네트워킹, 스토리지, 로그 관리 등을 쉽게 처리할 수 있도록 도와줍니다.

  • Dashboard

Istio

Istio는 Kubernetes에서 서비스 메시를 관리하는 오픈소스 플랫폼으로, 마이크로서비스 간의 트래픽 제어, 보안, 모니터링 등을 제공합니다. Istio Ingress Gateway는 외부 트래픽을 클러스터 내부로 라우팅하여 외부와의 연결을 효율적으로 제어합니다.

Kiali

Kiali는 Istio 서비스 메시의 시각적 관리를 위한 도구입니다. Kiali는 서비스 간의 트래픽 흐름을 시각적으로 보여주며, 서비스의 의존성 및 상태를 실시간으로 모니터링할 수 있습니다.

  • Dashboard

Jenkins

Jenkins는 지속적 통합(CI)과 지속적 배포(CD)를 자동화하는 오픈소스 도구입니다. 다양한 플러그인을 통해 빌드, 테스트, 배포 작업을 자동화하며, 소프트웨어 개발 프로세스를 빠르고 안정적으로 만들 수 있습니다.

  • Dashboard

ArgoCD

ArgoCD는 Kubernetes 네이티브의 지속적 배포(CD) 도구로, GitOps 방식으로 애플리케이션을 배포하고 관리합니다. Git 리포지토리를 기준으로 자동으로 배포와 동기화를 진행하여 애플리케이션의 상태를 관리합니다.

  • Dashboard

Tech Stack

이 프로젝트는 다양한 기술 스택을 활용하여 배포 및 관리되었습니다. 아래는 사용된 주요 기술들입니다.

Host OS & Virtual Machine

Host OS: Windows
프로젝트에서 호스트 운영 체제로 Windows OS를 사용하여 가상화 환경을 구축했습니다. Windows는 Hyper-V를 통해 가상 머신을 실행하는 데 적합한 환경을 제공합니다.

Virtual Machine: Hyper-V
Windows OS에서 Hyper-V를 사용하여 가상화 환경을 설정하고, 여러 가상 머신을 효율적으로 관리할 수 있었습니다. Hyper-V는 가상 머신을 실행하고 자원을 최적화하는 데 유용한 도구입니다.

Master & Worker Nodes: Rocky Linux
Rocky Linux는 프로젝트의 마스터 및 워커 노드 운영 체제로 사용되었습니다. 이 배포판은 안정성과 성능이 뛰어나 Kubernetes 클러스터 환경에서 최적의 성능을 발휘하며, 서버 및 클러스터 환경 운영에 적합합니다.

Kubernetes

Kubernetes는 컨테이너화된 애플리케이션을 자동으로 배포하고 관리하는 오픈 소스 시스템입니다. 이 프로젝트에서는 RKE2 (Rancher Kubernetes Engine 2)를 사용하여 Kubernetes 클러스터를 구축했습니다. RKE2는 안정성과 보안성을 강조한 Kubernetes의 경량화된 버전으로, 다양한 환경에서 쉽게 배포할 수 있습니다. 또한, Helm을 사용하여 애플리케이션을 Kubernetes 클러스터에 자동으로 배포하고 관리했습니다.

Helm은 Kubernetes의 패키지 관리 도구로, 애플리케이션을 Chart라는 템플릿으로 정의하여 배포하고 관리할 수 있습니다. Helm을 통해 복잡한 설정을 간단하게 패키징하여 Kubernetes 클러스터에 배포할 수 있으며, 다양한 환경에 맞춰 구성 파일을 쉽게 수정할 수 있습니다. 이 프로젝트에서는 Helm을 활용해 여러 애플리케이션을 효율적으로 관리하고, 반복 가능한 배포를 실현했습니다.

CI/CD Tools

  • Jenkins: CI/CD 파이프라인 자동화를 위한 Jenkins 사용. Jenkins는 애플리케이션 빌드, 테스트 및 배포를 자동화하여 개발자의 생산성을 높여줍니다.
  • ArgoCD: GitOps 기반의 지속적인 배포 및 관리 도구. ArgoCD는 Git 저장소를 배포 소스로 사용하여 Kubernetes 클러스터에 애플리케이션을 배포하고 관리하는 데 도움을 줍니다. GitOps 접근 방식으로 인프라를 코드화하여 더 나은 배포와 버전 관리를 실현할 수 있습니다.

Application Services

  • Cert Manager: SSL 인증서를 자동으로 관리하고 발급하는 도구입니다. Cert Manager는 Kubernetes 클러스터 내에서 인증서를 생성하고 갱신하며, 애플리케이션의 보안을 강화합니다.
  • Nginx: Nginx는 고성능 웹 서버로, 이 프로젝트에서는 웹 서버로 사용하여 HTTP 요청을 처리하고, 정적 파일을 제공하는 역할을 했습니다. Nginx는 효율적인 트래픽 처리와 로드 밸런싱을 지원하며, 다양한 웹 애플리케이션에 대한 요청을 처리할 수 있습니다.
  • Rancher: Kubernetes 클러스터 관리 및 오케스트레이션 도구입니다. Rancher는 여러 Kubernetes 클러스터를 중앙에서 관리할 수 있도록 하여, Kubernetes 환경을 보다 쉽게 관리하고 모니터링할 수 있도록 도와줍니다.

Service Mesh

  • Istio: Istio는 서비스 메쉬로, 서비스 간의 트래픽 관리, 보안, 모니터링, 트랜잭션 추적을 제공하는 도구입니다. 이 프로젝트에서는 Istio를 사용하여 마이크로서비스 간의 통신을 보다 안전하고 효율적으로 관리했습니다.
  • Istio Ingress Gateway: Istio의 Ingress Gateway를 사용하여 Kubernetes 클러스터에 외부 트래픽을 안전하게 라우팅하고, SSL/TLS 인증서를 설정하여 보안성을 강화했습니다. 이를 통해 외부 요청을 서비스로 전달하는 데 있어 더 높은 제어와 보안성을 제공하였습니다.

Monitoring

  • Kiali: Istio 기반의 서비스 트래픽 모니터링 도구입니다. Kiali는 Istio의 메트릭스, 로그 및 트래픽 흐름을 시각화하여 클러스터 내의 서비스 간 상호작용을 명확하게 보여줍니다. 이를 통해 문제를 쉽게 파악하고 해결할 수 있습니다.
  • K9S: Kubernetes 클러스터의 상태를 실시간으로 모니터링하고 관리할 수 있는 CLI 도구입니다. K9S는 사용자에게 직관적인 터미널 기반 인터페이스를 제공하여, Kubernetes 자원의 상태를 쉽게 파악하고, 필요에 따라 자원에 대한 액션을 빠르게 실행할 수 있습니다. 이 도구는 클러스터의 효율적인 관리와 디버깅에 매우 유용합니다.

이 기술 스택들을 사용하여 로컬 VM 환경에서 Kubernetes 클러스터를 구축하고, 다양한 DevOps 툴을 통해 CI/CD 파이프라인을 자동화하며, Kubernetes 환경에서 애플리케이션을 안정적으로 배포하고 관리할 수 있었습니다. 이러한 기술들은 효율적인 배포와 안정적인 운영을 가능하게 만들어, 프로젝트의 성공적인 진행에 큰 기여를 했습니다.
